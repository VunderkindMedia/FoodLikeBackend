## Добавление/обновление пользователя.

Метод позволяет добавить пользователя в базу для дальнейшего использования его токена при отправке push уведомлений.
Если пользователь с переданным номером телефона уже есть в базе, он может быть обновлен (отредактирован с изменением полей `os` и `token`. Например, когда клиент купил новое устройство) 

Строка запроса:  
`GET api/clients/add`

### Параметры
|Параметр|Описание|Примечания|
| :-------------: |:-------------:| :----:|
|`client_phone` | номер телефона в формате (+79999999999) - 12 символов | *[обязательное поле]* |
|`token`       | Токен устройства для отправки PUSH уведомлений      | *[обязательное поле]* |
|`os`           | Операционная система (android / iOS)     | *[обязательное поле]* |
|`will_rewrite` |В положении false запрещает перезапись данных пользователя, разрешает только добавить нового.|*необязательное поле (по-умолчанию: **true*** | 

### Пример запроса
`GET api/clients/add?client_phone=+79990702842&token=dgadsg78asd9g87a9d7g9dfgdf&os=ios`

```javascript
{
    client_phone: '+79999999999'
    token: 'dgadsg78asd9g87a9d7g9dfgdf'
    os: 'ios'
}
```

### Пример ответа
`200 OK`

```json
{
    "data": {
        "msg": "Client is updated!",
        "phone": "+79999999999",
        "token": "dgadsg78asd9g87a9d7g9dfgdf",
        "os": "ios"
    }
}
```
